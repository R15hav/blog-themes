# CLAUDE.md

This file provides guidance to Claude Code when working in this repository.

## Project overview

This is the **theme repository** for the blog platform. It contains only CSS/SCSS ŌĆö no JavaScript, no framework. The compiled `themes/default.css` is served as a static asset via jsDelivr CDN and injected as a `<link>` tag into all three Next.js route groups (`(blog)`, `(auth)`, `(admin)`) at runtime.

The frontend lives in a separate repo (`blog/frontend/`). Changes here take effect without redeploying the frontend.

## Commands

```bash
npm install        # install sass (one-time)
npm run build      # compile SCSS ŌåÆ default.css (run after any edit)
npm run watch      # compile on save during development
```

To compile a non-default theme:
```bash
npx sass themes/dark.scss themes/dark.css --style=compressed --no-source-map
```

## Critical rules

- **Never edit `themes/default.css` or `themes/dark.css` directly.** They are generated by `npm run build`. Manual edits will be overwritten on the next build.
- **Always run `npm run build` after editing any partial.** The committed `.css` files must stay in sync with the SCSS source.
- **`_variables.scss` is the only file with hardcoded values.** All other partials use `var(--token)`.
- One partial per concern ŌĆö do not add unrelated rules to an existing partial.

## Token system

All design tokens are CSS custom properties in `_variables.scss`. The naming convention is `--color-*`, `--font-*`, `--text-*`, `--space-*`, `--radius-*`, `--shadow-*`, `--transition-*`.

The **same token names** are used in the frontend's `globals.css` files (admin, auth, blog). A theme override to `--color-bg` changes the background across all three route groups simultaneously.

### Key tokens

| Token | Purpose |
|---|---|
| `--color-bg` | Page background |
| `--color-bg-subtle` | Sidebar, card, alt row backgrounds |
| `--color-bg-muted` | Hover/active tints |
| `--color-border` | Default rule/divider |
| `--color-border-soft` | Subtle secondary divider |
| `--color-text` | Primary text |
| `--color-text-2` | Secondary text |
| `--color-text-3` | Tertiary/muted text |
| `--color-text-4` | Ultra-muted labels/caps |
| `--color-accent` | Brand/CTA color |
| `--color-accent-bg` | Tinted accent background |
| `--color-accent-text` | Darker accent ŌĆö readable on `--color-bg` |
| `--color-accent-fg` | Text ON a filled accent surface |
| `--color-ok` / `--color-ok-text` / `--color-ok-bg` | Success states |
| `--color-warn` / `--color-warn-text` / `--color-warn-bg` | Warning states |
| `--color-error` | Error text |

## How the theme reaches the browser

1. `themes/default.css` is committed and pushed to GitHub
2. jsDelivr serves it at `https://cdn.jsdelivr.net/gh/USER/REPO@BRANCH/themes/default.css`
3. Admin registers that URL in the blog admin ŌåÆ Theme Management page
4. The blog backend stores the URL; `GET /api/v1/theme/active` returns it
5. All three Next.js layouts (`(blog)`, `(auth)`, `(admin)`) inject it via `<link rel="stylesheet">`
6. The CDN stylesheet loads **after** `globals.css`, so its rules take precedence

## Writing a new theme

**Layer 1 ŌĆö Token override (covers ~90% of use cases):**

```css
/* mytheme.css */
:root {
  --color-bg:          #0f0f0f;
  --color-bg-subtle:   #1a1a1a;
  --color-bg-muted:    #242424;
  --color-border:      #2e2e2e;
  --color-border-soft: #252525;
  --color-text:        #e8e8e8;
  --color-text-2:      #b0b0b0;
  --color-text-3:      #6e6e6e;
  --color-text-4:      #4a4a4a;
  --color-accent:      #e94560;
  --color-accent-bg:   rgba(233,69,96,0.14);
  --color-accent-text: #f06080;
  /* ... any of the ~35 tokens */
}
```

**Layer 2 ŌĆö Component override (advanced):**

```css
/* add after :root */
header {
  backdrop-filter: blur(8px);
  background: color-mix(in srgb, var(--color-bg), transparent 15%);
}
```

Copy `dark.scss` as a starting point and run `npx sass themes/dark.scss themes/dark.css --style=compressed --no-source-map`.

## SCSS conventions

- **SCSS nesting** used throughout to group related rules
- **No SCSS variables** ŌĆö all tokens are CSS custom properties in `:root`
- **No mixins or functions** ŌĆö plain readable SCSS
- Partials use `_name.scss`; `default.scss` imports them with `@use 'scss/name'`
